# Vue Component & Pages Architecture

- **File Naming:**
  - Folders: `kebab-case`
  - Components: `PascalCase`

  _Keep general components in the `app/components/` folder, and project-specific components in their own nested `/components/` folders inside pages directories._


  - **Prefixing Conventions**:
    - Nuxt UI components are prefixed `U` (e.g., `<UButton>`).
    - RimelightComponents are prefixed `RC` (e.g., `<RCMovieCard>`).
    - Project-specific components use their own unique prefix (e.g., `ID` for idantity.me) to avoid collisions.

  _These are automatically generated by the Nuxt configuration, files should not manually include them._


- **SFC Structure:**
  - All `.vue` files MUST include exactly one of the following blocks, even if empty:
    1. `<script setup lang="ts">`
    2. `<template>`
    3. `<style scoped>`


- **Region Formatting:**
  - All `.vue` files MUST be divided into region comments (`/* region RegionName */` and `/* endregion */`) in the `<script setup>` block in the exact order below.
  - **Components (`/components/`)**: Props, Emits, Slots, Styles, State, Meta, Lifecycle, Logic.
  - **Pages (`/pages/`)**: State, Meta, Lifecycle, Logic.


- **Explicit Formats:**
  - **Props:** When existent, components MUST export props as an interface and use destructuring through `defineProps<{ ... }>()`. Example:
    ```ts
    /* region Props */
    export interface CardProps {
      title: string
    }
    const { title = 'Default Title' } = defineProps<CardProps>()
    /* endregion */
    ```
  - **Emits:** When existent, components MUST export emits as an interface and use destructuring through `defineEmits<{ ... }>()` with tuple syntax. Example:
    ```ts
    /* region Emits */
    export interface UCardEmits {
      submit: [payload: { id: string }]
    }
    const emit = defineEmits<UCardEmits>()
    /* endregion */
    ```
  - **State:** Use `useState()` for shared state, `ref()` for local state. Example inside `/* region State */`:
    ```ts
    /* region State */
    const isModalOpen = ref(false)
    /* endregion */
    ```
  - **Slots:** When existent, components MUST export slots as an interface and implement them using `defineSlots<{ ... }>()`. Example:
    ```ts
    /* region Slots */
    export interface UCardSlots {
      default(props?: {}): any
      header(props?: {}): any
      footer(props?: {}): any
    }
    const slots = defineSlots<UCardSlots>()
    /* endregion */
    ```
